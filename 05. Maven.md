[Maven]
	0. 정의
	  - Maven은 프로젝트 객체 모델 (Project Object Model), 표준의 집합, 프로젝트 라이프사이클, 의존성 관리 시스템, 라이프사이클에 정의된 단계에서 플러그인 목표를 실행하는 논리를 포함하는 프로젝트 관리 도구이다.
	    Maven을 사용할 때 잘 정의된 프로젝트 객체 모델을 사용한 프로젝트를 기술하며, Maven이 그 다음에 공유된(혹은 사용자가 만든) 플러그인들로부터 cross-cutting 로직을 적용할 수 있다.
	    
	
	1. Maven을 사용하는 이유
	  - 프로젝트의 빌드에 필요한 것이 무엇인가
	  - 다운로드 받을 필요가 있는 라이브러리가 어떤 것인가.
	  - 다운로드 받은 것을 어디에 넣어야 하는가
	  - 빌드에서 어떤 goal을 실행할 수 있는가
	  
	
	2. Maven을 사용하기 전 사용했던 툴은 Ant이다.
	
	3. 동작원리
	  - 개발자 A가 'tuto'라는 라이브러리를 만들었다.
	  - 만든 라이브러리를 Maven툴을 이용해서 서버에 업로드한다.
	  - 개발자 B,C가 'tuto' 라이브러리가 필요할 때, 'pom.xml'의 설정을 통해 자동으로 maven툴을 사용하여 서버에 접속하여 라이브러리를 다운받고, 자신의 프로젝트에 자동으로 추가한다.
	  - 필요에따라 'pom.xml'파일을 설정하면 Maven 툴은 주기적으로 서버와 통신하여 최신버전의 'tuto'를 다운받게 할 수 있고, 개발자 A를 포함하여 다른개발자가 'tuto'라는 라이브러리를 수정하여 서버에 업로드 할 수 있다.
	  - 'tuto'를 사용하는 'tuto2'라는 라이브러리가 있을 경우 Maven은 알아서 의존성을 체크하여 두 라이브러리를 다운받는다.
